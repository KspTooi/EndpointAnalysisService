# BioCode 数据库设计 · 数据域规范

> 本文档定义了 BioCode 项目数据库设计中允许使用的**全部数据域（Domain）**。
> 设计人员在 PDMan 中创建字段时，**必须**从下列数据域中选择，禁止直接使用原生数据库类型。

---

## 一、设计原则

1. **语义优先**：字段的"意义"决定数据域，而非"长度"。优先选择强语义域（如 `GP手机号` 而非 `BI业务用名`）。
2. **短路选择**：从强语义域（G/M/P/L）开始判断，都不匹配再回落到通用域（BI/C/R/N/D）。
3. **最小够用**：选择能容纳预期最大值的**最小档位**，不要为"万一"预留。
4. **禁止扩展**：未经架构评审，禁止新增数据域，禁止修改已有域的长度/类型。
5. `**S默认串`**：**仅用于原型期占位，正式表必须替换为具体语义域。

---

## 二、数据域全量清单（43 个）

### 1. 业务名称类（BI · Business Identify）

用于**非人员、非编码**的业务对象名称。


| 数据域代码   | 显示名称    | 典型场景                   |
| ------- | ------- | ---------------------- |
| `S`     | 默认串     | **仅原型期占位**，正式表禁用       |
| `N`     | 姓名（通用）  | 兼容旧数据，新设计请用 `GN`/`GNG` |
| `BI`    | 小型业务用名  | 商品名、部门名、角色名            |
| `BIM`   | 中型业务用名  | 企业全称、短标题               |
| `BIL`   | 大型业务用名  | 复杂资源名、完整长标题            |
| `BIXL`  | 超大型业务用名 | 极长复合名称（慎用）             |
| `BI2XL` | 巨型业务用名  | 极端长度标识（慎用，考虑拆字段）       |


---

### 2. 特殊语义类（G · General）

**强语义**字段，锁死长度与用途，禁止混用。


| 数据域代码 | 显示名称      | 典型场景               |
| ----- | --------- | ------------------ |
| `GN`  | 人员姓名（中国）  | 中文姓名，最长 10 汉字      |
| `GNG` | 人员姓名（国际化） | 国际化姓名、复合名、带中间名     |
| `GP`  | 手机号       | 手机号、固话（含国家码）       |
| `GE`  | 邮箱        | 电子邮箱地址             |
| `GUM` | 中型 URL/链接 | 常规 URL、资源链接        |
| `GUL` | 大型 URL/链接 | 带长查询参数的 URL        |
| `GIP` | IP 地址     | IPv4/IPv6 地址（含分隔符） |
| `GH`  | 哈希值       | SHA256、MD5、签名摘要、令牌 |


---

### 3. 业务编码类（C · Code）

用于**机读**的标识符，通常全大写字母+数字+分隔符。


| 数据域代码 | 显示名称  | 典型场景                       |
| ----- | ----- | -------------------------- |
| `CS`  | 小型编码  | 分类代码、短缩写、字典 Key            |
| `CM`  | 中型编码  | **最常用**：单据号 `ORD-2023-001` |
| `CL`  | 大型编码  | 外部系统冗余 ID、复合流水号            |
| `CXL` | 超大型编码 | 超长第三方对接标识                  |


---

### 4. 备注文本类（R · Remark）

用于**人读**的描述性文本。


| 数据域代码 | 显示名称  | 典型场景           |
| ----- | ----- | -------------- |
| `RS`  | 小型备注  | 字段说明、单行审批意见    |
| `RM`  | 中型备注  | 多行说明、商品简介、任务摘要 |
| `RL`  | 大型备注  | 日志内容、文章摘要段落    |
| `RXL` | 超大型备注 | 富文本 HTML、网页源码  |


---

### 5. 整数类（N · Number）


| 数据域代码  | 显示名称  | 典型场景          |
| ------ | ----- | ------------- |
| `NS`   | 小型整数  | 状态码、类型枚举、性别代码 |
| `NM`   | 中型整数  | 常规计数、库存数量     |
| `NL`   | 大型整数  | 自增主键、海量统计值    |
| `NSEQ` | 排序型整数 | 列表/树的 `seq`   |


---

### 6. 浮点/精确小数类（D · Decimal）

**非金额**的数值型字段。


| 数据域代码 | 显示名称  | 典型场景           |
| ----- | ----- | -------------- |
| `DS`  | 小型浮点  | 费率、折扣率、税率、比例系数 |
| `DM`  | 中型浮点  | 重量、长度、常规度量值    |
| `DL`  | 大型浮点  | 高精度度量、科学测量值    |
| `DXL` | 超大型浮点 | 科学计算、超高精度聚合值   |


---

### 7. 金额类（M · Money）

**专用金额**字段，与 D 系列严格区分。**所有涉及钱的字段必须使用 M 系列。**


| 数据域代码 | 显示名称 | 典型场景                        |
| ----- | ---- | --------------------------- |
| `MM`  | 中型金额 | **99% 场景选这个**：订单金额、单价、工资、余额 |
| `ML`  | 大型金额 | 汇率、利息计算、加密货币、大宗交易           |


---

### 8. 日期/时间类（L · Local time）


| 数据域代码 | 显示名称         | 典型场景             |
| ----- | ------------ | ---------------- |
| `LD`  | 日期（无时间）      | 生日、合同生效日、账期      |
| `LDT` | 日期与时间        | 创建时间、更新时间、业务发生时间 |
| `LDU` | 毫秒级 Unix 时间戳 | 跨语言/跨时区交互、精确到毫秒  |


---

### 9. 主键类（P · Primary）

**主键专用**，禁止用于普通字段。


| 数据域代码 | 显示名称  | 典型场景              |
| ----- | ----- | ----------------- |
| `P`   | 雪花 ID | 分布式发号器主键（18/19 位） |
| `PU`  | UUID  | 去横线 32 位 UUID 主键  |


---

### 10. JSON 与 UI 语义类（J · JSON）

**J 系列不仅是存储声明，更是 UI 渲染指令**——前端根据数据域自动渲染对应组件。


| 数据域代码   | 显示名称    | 典型场景 / UI 行为        |
| ------- | ------- | ------------------- |
| `JSON`  | 纯 JSON  | 通用非结构化数据            |
| `JFAJ`  | 多文件附件   | 前端渲染"拖拽上传 / 附件列表"组件 |
| `JEFAJ` | 扩展多文件附件 | 前端渲染带元数据的复杂附件管理组件   |
| `JCTJ`  | 扩展多标签   | 前端渲染动态 Tag 标签输入组件   |


---

## 三、选型决策树

```
字段是什么？
├─ 主键         → P（雪花 ID）/ PU（UUID）
├─ 钱           → MM（常规）/ ML（大额/高精度）
├─ 时间
│   ├─ 只到天   → LD
│   ├─ 到秒     → LDT
│   └─ 到毫秒/跨时区 → LDU
├─ 整数         → NS / NM / NL / NSEQ（排序号）
├─ 小数（非钱） → DS / DM / DL / DXL
├─ 姓名
│   ├─ 中国人   → GN
│   └─ 国际化   → GNG
├─ 强语义串     → GP手机 / GE邮箱 / GUM/GUL链接 / GIP / GH哈希
├─ 编码（机读） → CS / CM / CL / CXL
├─ 备注（人读） → RS / RM / RL / RXL
├─ 业务名       → BI / BIM / BIL / BIXL / BI2XL
├─ JSON         → JSON / JFAJ / JEFAJ / JCTJ
└─ 都不是       → （原型期）S，正式表禁用
```

## 四、常见混淆场景


| 场景    | ❌ 错误选型                    | ✅ 正确选型                        | 原因                     |
| ----- | ------------------------- | ----------------------------- | ---------------------- |
| 订单金额  | `DM(16,2)`                | `MM(14,4)`                    | 金额必须用 M 系列，保 4 位防误差    |
| 手机号   | `CM(32)` 或 `BI(80)`       | `GP(20)`                      | 强语义域优先                 |
| 汇率    | `DS(10,4)`                | `ML(22,8)`                    | 汇率是金额类高精度场景            |
| 是否删除  | `NS` TINYINT              | `LDT`                         | 使用删除时间                 |
| 排序号   | `NM` INT                  | `NSEQ` INT                    | 强语义，便于代码生成器识别          |
| 创建时间  | `NL` BIGINT               | `LDT`                         | 时间不用裸 BIGINT           |
| 主键 ID | `NL` BIGINT               | `P` BIGINT 或 `PU` VARCHAR(32) | 主键用 P 系列               |
| 商品描述  | `BIXL(1000)`              | `RM(2000)`                    | "描述"用 R 系列，"名称"用 BI 系列 |
| 图片列表  | `RXL` LONGTEXT 存 JSON 字符串 | `JEFAJ`                       | 附件必须用 J 系列驱动 UI        |


---

## 五、禁止事项

1. **禁止**直接使用 `VARCHAR(N)`、`INT`、`DECIMAL(...)` 等原生类型声明字段。
2. **禁止**修改本规范中任何数据域的长度或类型。如有刚需，走架构评审流程。
3. **禁止**在正式表中使用 `S默认串`。
4. **禁止**用 `NS` 表示布尔语义，必须用 `B`。
5. **禁止**用 `DM`/`DS` 存储金额，必须用 `MM`/`ML`。
6. **禁止**用 `NL` 裸 `BIGINT` 存储时间戳，必须用 `LDU`。
7. **禁止**用 `RXL` 或 `JSON` 字符串化存储附件列表，必须用 `JFAJ`/`JEFAJ`。
