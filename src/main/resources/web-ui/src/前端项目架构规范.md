# 前端项目架构规范

本项目采用 **SOA架构**，按业务领域分类组织代码。

## 目录结构

```
src/
├── soa/                    # 全局SOA服务（跨业务复用）
│   └── console-framework/
└── views/                  # 业务模块（按领域划分）
    ├── drive/
    ├── relay/
    └── requestdebug/
```

## 业务模块内部结构

每个业务模块包含以下目录：

| 目录            | 职责                                       |
| :-------------- | :----------------------------------------- |
| **api/**        | 接口请求、DTO/VO/PO 类型定义               |
| **components/** | 业务组件，只负责渲染                       |
| **service/**    | 业务逻辑、状态管理                         |
| **入口文件**    | 如 `Drive.vue`，组装 Service 和 Components |

## 全局 vs 局部

| 位置              | 放什么                 |
| :---------------- | :--------------------- |
| `src/soa/xxx/`    | 跨业务复用的服务组件   |
| `src/views/xxx/`  | 只在当前业务使用的代码 |
| `src/commons/`    | 公共工具、实体定义     |
| `src/components/` | 纯UI组件，不含业务逻辑 |

## 命名规范

| 类型    | 规范       | 示例                                 |
| :------ | :--------- | :----------------------------------- |
| 文件夹  | kebab-case | `console-framework`, `request-debug` |
| Vue组件 | PascalCase | `DriveEntryGrid.vue`                 |
| TS文件  | PascalCase | `DriveService.ts`, `DriveApi.ts`     |
